# Frontend Brief

이 문서는 인터랙티브 홈페이지 제작자를 위한 구현 가이드입니다.

## 참고 방향

- 참고 저장소: <https://github.com/whchoi98/ontology-for-assembly>
- 참고 사이트: <https://assembly.whchoi.net/>

참고 사이트처럼 좌측 탐색 또는 상단 탭, KPI 카드, 시나리오 카드, 상세 패널을 조합하는 구조가 잘 맞습니다. 이 프로젝트에서는 “시나리오”를 섹터와 정책 카드로 치환하면 됩니다.

중요한 방향은 “분석을 아는 사람만 이해하는 대시보드”가 아니라 “처음 방문한 독자도 정책, 뉴스, 주식 흐름의 관계를 차례대로 따라갈 수 있는 결과 페이지”입니다. 그래서 첫 화면에서는 숫자를 많이 나열하기보다, 분석 질문과 핵심 KPI를 함께 보여주는 구성이 좋습니다.

## 첫 화면 KPI

| KPI | 값 |
| --- | --- |
| 정책 | 50건 |
| 뉴스 | 121,954건 |
| 섹터 | 5개 |
| 평균 정합 | r=0.706 |
| 방향 정합 | 50/50 |
| 강한·높은 정합 | 38/50 |

첫 화면에는 KPI만 놓지 말고, 바로 위나 아래에 한 문장 설명을 붙이는 것을 권장합니다.

```text
정책 뉴스의 NLP 반응흐름과 관련 주식그룹의 초과수익 CAR가 정책별로 얼마나 함께 움직였는지 비교했습니다.
```

KPI는 아래처럼 묶으면 처음 보는 사람이 이해하기 쉽습니다.

| 묶음 | 표시할 값 | 이유 |
| --- | --- | --- |
| 분석 규모 | 정책 50건, 뉴스 121,954건, 섹터 5개 | 분석 대상이 충분히 구조화되어 있음을 보여줍니다. |
| 핵심 결과 | 방향 정합 50/50, 평균 r=0.706 | 전체 결과를 빠르게 전달합니다. |
| 해석 안전장치 | 강한·높은 정합 38/50, 약한 정합 4/50 | 모든 정책이 강하게 맞았다는 오해를 막습니다. |

## 정책 카드 필드

`policy_alignment_status`를 사용합니다.

- `policy_code`
- `policy_name`
- `sector`
- `display_label`
- `lag`
- `best_group`
- `r`
- `alignment_status`
- `alignment_grade`
- `n_ptei`

권장 카드 문구:

```text
P08 K칩스법 1차
반도체 · LLM 시장반응 신호 · lag 0일
방향 정합 · 강한 정합 · r=0.942
선택 주식그룹: KOSDAQ·대기업
기사수: 2,232
```

`r < 0.4` 정책은 방향 정합은 표시하되, 강한 성과 카드처럼 보이지 않게 `약한 정합` 배지로 구분하세요.

정책 카드에는 가능하면 짧은 보조 설명을 붙이세요.

| 필드 | 보조 설명 예시 |
| --- | --- |
| 정책명 | 분석 대상 정책입니다. |
| 섹터 | 정책이 주로 관련되는 산업 묶음입니다. |
| NLP 신호 | 뉴스 흐름을 숫자로 바꾼 기준입니다. |
| lag | 뉴스와 주가 흐름을 비교할 때 적용한 시간 간격입니다. |
| best_group | 여러 주식그룹 중 해당 정책과 가장 잘 맞은 그룹입니다. |
| r | 뉴스 흐름과 CAR 흐름의 유사도입니다. |
| alignment_grade | r 값에 따라 나눈 정합 강도입니다. |

카드 정렬 옵션은 `정합 강도순`, `정책 번호순`, `기사 수순`, `섹터별` 정도면 충분합니다. 낮은 r 정책을 숨기기보다 필터로 확인할 수 있게 두는 편이 결과 신뢰에 도움이 됩니다.

## 초심자용 문구와 툴팁

홈페이지에는 데이터 분석에 익숙하지 않은 독자를 위해 짧은 도움말을 붙이는 것을 권장합니다.

| 위치 | 문구 |
| --- | --- |
| `r` 툴팁 | 뉴스 흐름과 주가 흐름이 얼마나 비슷하게 움직였는지를 나타내는 값입니다. 1에 가까울수록 강합니다. |
| `방향 정합` 툴팁 | 두 흐름이 같은 방향으로 잡혔다는 뜻입니다. 정합 강도는 r 등급을 함께 봐야 합니다. |
| `약한 정합` 툴팁 | 방향은 같지만 r 값이 낮아 강한 사례로 볼 수 없습니다. |
| `CAR` 툴팁 | 시장 전체 움직임을 뺀 관련 주식그룹의 누적 초과수익입니다. |
| `lag` 툴팁 | 뉴스 흐름과 주가 흐름을 비교할 때 적용한 시간 간격입니다. |

처음 보는 독자에게는 `정책별 방향 정합 50/50`과 `강한·높은 정합 38/50`을 함께 보여주세요. 이렇게 해야 “모든 정책이 강하게 맞았다”는 오해를 줄일 수 있습니다.

## 상세 차트

`plot_series`를 사용합니다.

1. 선택된 `policy_n`으로 필터링
2. `kind === "nlp"`와 `kind === "car"`를 분리
3. `cal_day`를 x축으로 정렬
4. 이중축 라인 차트로 표시

권장 범례:

- 파란 선: NLP 누적흐름
- 빨간 선: 정책주 초과수익 CAR
- x축: 정책일 D0 기준 상대일

상세 차트 위에는 아래 설명을 붙이면 좋습니다.

```text
파란 선은 정책 관련 뉴스의 누적 반응흐름, 빨간 선은 관련 주식그룹의 누적 초과수익 흐름입니다. 두 선의 절대 크기보다 시간에 따른 방향과 모양을 비교해 보세요.
```

차트 하단에는 선택 정책의 해석 문장을 자동으로 만들 수 있습니다.

```text
이 정책은 {sector} 섹터에 속하며, {display_label} 기준으로 {best_group} CAR와 비교했습니다. 정합 상관계수는 r={r}이며, 등급은 {alignment_grade}입니다.
```

`alignment_grade`가 `약한 정합`이면 아래 문구를 추가하세요.

```text
이 정책은 방향은 같지만 r 값이 낮아 강한 정합 사례로 해석하지 않습니다.
```

## 데이터 로딩 예시

```ts
type PlotPoint = {
  policy_n: number;
  kind: "nlp" | "car";
  cal_day: number;
  value: number;
};

const payload = await fetch("/data/processed/web_payload.json").then((res) => res.json());
const policy = payload.policy_alignment_status.find((row) => row.policy_code === "P08");
const series = payload.plot_series
  .filter((row: PlotPoint) => row.policy_n === policy.policy_n)
  .sort((a: PlotPoint, b: PlotPoint) => a.cal_day - b.cal_day);
```
